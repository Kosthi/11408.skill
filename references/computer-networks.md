## 考纲知识框架

### 第一章 计算机网络体系结构（考频：中）

**高频考点：**
1. **OSI七层 vs TCP/IP四层 vs 五层参考模型对照**
   - 物理层 → 数据链路层 → 网络层 → 传输层 → 应用层
   - 每层的PDU名称：比特、帧、分组/包、报文段、报文

2. **协议三要素**：语法、语义、同步（时序）

3. **电路交换 vs 分组交换 vs 报文交换**
   - 分组交换的优势：资源利用率高、延迟低
   - 存储转发延迟计算

**易错点：**
- TCP/IP模型没有会话层和表示层
- 五层参考模型是教学用的，实际使用TCP/IP四层

### 第二章 物理层（考频：低）

**常考知识点：**
- 奈奎斯特定理（无噪声）：C = 2W × log₂V
- 香农定理（有噪声）：C = W × log₂(1 + S/N)
- 编码方式：NRZ、曼彻斯特编码、差分曼彻斯特编码
- 信道复用：FDM、TDM、WDM、CDM（码分复用的计算）

**易错点：**
- 信噪比dB的转换：dB = 10 × log₁₀(S/N)
- 实际数据率取奈奎斯特和香农结果中的较小值
- 曼彻斯特编码的波特率 = 2 × 比特率

---

### 第三章 数据链路层（考频：高）

**高频考点：**
1. **差错控制**（选择题高频）
   - 奇偶校验：只能检1位错
   - CRC循环冗余校验：能检所有奇数位错，生成多项式的应用
   - 海明码：纠1位错检2位错，校验位数 2^r ≥ k+r+1

2. **流量控制与可靠传输**（大题考点）
   - 停止等待协议：效率 = T_f / (T_f + RTT + T_ack)
   - 回退N帧(GBN)：发送窗口 ≤ 2^n - 1
   - 选择重传(SR)：发送窗口 = 接收窗口 ≤ 2^(n-1)
   - 信道利用率计算

3. **介质访问控制**
   - CSMA/CD（以太网）：争用期 = 2τ，最小帧长 = 2τ × 数据率
   - CSMA/CA（无线LAN）
   - 以太网帧格式、MAC地址

4. **交换机和局域网**
   - 交换机工作在数据链路层
   - VLAN的原理

**易错点：**
- GBN发送窗口不能等于 2^n（否则新旧帧混淆）
- SR发送窗口和接收窗口都不能超过 2^(n-1)
- CRC只能检错不能纠错（在数据链路层的应用中）
- 最小帧长 = 2 × 传播延迟 × 数据传输速率

### 第四章 网络层（考频：高）

**高频考点：**
1. **IP地址与子网划分**（大题高频）
   - A/B/C类地址范围和默认掩码
   - 子网划分：子网掩码的计算
   - CIDR无类域间路由：地址聚合/超网
   - 特殊IP地址：网络地址、广播地址、环回地址
   - 子网中可用主机数 = 2^主机位 - 2

2. **IP数据报格式**
   - 首部固定20字节 + 可选项
   - 总长度、标识、标志(MF/DF)、片偏移
   - TTL、协议字段、首部校验和

3. **IP数据报分片**（选择题/大题）
   - 最大传输单元MTU
   - 片偏移以8字节为单位
   - 分片计算过程

4. **路由算法**
   - 静态路由 vs 动态路由
   - RIP（距离向量，最大跳数15）
   - OSPF（链路状态，Dijkstra算法）
   - BGP（路径向量，AS间路由）

5. **ARP协议**
   - IP地址 → MAC地址的映射
   - ARP请求广播，ARP应答单播

6. **ICMP协议**
   - 差错报告报文和询问报文
   - ping和traceroute

7. **NAT和DHCP**
   - NAT：私有IP和公有IP的转换
   - DHCP：动态分配IP地址

**易错点：**
- 子网划分时网络地址和广播地址不可分配给主机
- 片偏移字段的值 = 实际偏移量 / 8
- RIP只适用于小型网络（跳数 ≤ 15，16表示不可达）
- ARP工作在网络层（虽然涉及MAC地址）

### 第五章 传输层（考频：极高）

**高频考点：**
1. **TCP vs UDP**（选择题必考）
   | 特性 | TCP | UDP |
   |------|-----|-----|
   | 连接 | 面向连接 | 无连接 |
   | 可靠性 | 可靠传输 | 不可靠 |
   | 流量控制 | 有（滑动窗口） | 无 |
   | 拥塞控制 | 有 | 无 |
   | 首部开销 | 20字节 | 8字节 |
   | 传输方式 | 面向字节流 | 面向报文 |

2. **TCP三次握手与四次挥手**（大题考点）
   - 三次握手：SYN → SYN+ACK → ACK
   - 四次挥手：FIN → ACK → FIN → ACK
   - 为什么需要三次握手（防止旧连接请求导致资源浪费）
   - TIME_WAIT状态持续2MSL

3. **TCP可靠传输机制**
   - 序号、确认号、超时重传
   - 滑动窗口机制
   - 快速重传（收到3个重复ACK）

4. **TCP拥塞控制**（大题高频）
   - 慢开始：cwnd从1开始指数增长
   - 拥塞避免：达到ssthresh后线性增长
   - 快重传：3个重复ACK → 立即重传
   - 快恢复：ssthresh = cwnd/2，cwnd = ssthresh
   - 画cwnd变化曲线图

5. **TCP流量控制**
   - 接收窗口rwnd
   - 发送窗口 = min(cwnd, rwnd)

**易错点：**
- 三次握手中SYN报文消耗一个序号，ACK报文不消耗序号
- 拥塞控制中慢开始的"慢"是指初始窗口小，增长速度其实是指数级
- 发送窗口取cwnd和rwnd的较小值
- 超时重传时ssthresh = cwnd/2，cwnd重置为1（回到慢开始）

### 第六章 应用层（考频：中高）

**高频考点：**
1. **DNS**
   - 域名解析过程：递归查询 vs 迭代查询
   - 域名服务器层次：根、顶级域、权威、本地

2. **HTTP**
   - 请求/响应报文格式
   - 持久连接 vs 非持久连接
   - HTTP/1.1 vs HTTP/2.0
   - Cookie和Session

3. **SMTP和POP3/IMAP**
   - 邮件发送和接收过程
   - SMTP用于推送，POP3/IMAP用于拉取

4. **FTP**
   - 控制连接(21端口) vs 数据连接(20端口)
   - 主动模式 vs 被动模式

**易错点：**
- DNS使用UDP(53端口)进行查询，区域传送用TCP
- HTTP默认端口80，HTTPS默认443
- SMTP只能传送7位ASCII，MIME扩展支持多媒体

## 大题解题模板

### 子网划分题
```
1. 确定需要的子网数或每个子网的主机数
2. 计算需要的子网位数或主机位数
3. 写出子网掩码
4. 列出各子网的网络地址、广播地址、可用主机范围
```

### TCP拥塞控制题
```
1. 明确初始cwnd=1、初始ssthresh值
2. 逐个RTT计算cwnd：
   - cwnd < ssthresh：慢开始，每RTT翻倍
   - cwnd ≥ ssthresh：拥塞避免，每RTT加1
3. 遇到超时：ssthresh = cwnd/2，cwnd = 1
4. 遇到3个重复ACK：ssthresh = cwnd/2，cwnd = ssthresh（快恢复）
5. 画出cwnd-RTT曲线图
```
